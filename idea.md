# Hostel Room Allocation System

## 📌 Project Overview
The Hostel Room Allocation System is a full-stack web application that manages hostel room allocation for students in an institution.

The system allows students to apply for hostel accommodation, and administrators can assign rooms based on availability and allocation strategy.

The main focus of this project is backend design using OOP principles and clean architecture patterns.

---

## 🎯 Scope of the Project

- Student registration and login
- Hostel and room management
- Room availability tracking
- Room allocation logic
- Admin approval and assignment
- Allocation history tracking
- Role-based access (Student/Admin)

---

## 🧠 Key Backend Features

- Encapsulation of core entities (Student, Room, Hostel)
- AllocationService handling business logic
- Strategy Pattern for room allocation logic
- Clean layered architecture:
  - Controller Layer
  - Service Layer
  - Repository Layer
  - Model Layer
- Proper exception handling
- Validation logic
- RESTful APIs

---

## 🏗 OOP Concepts Used

- Encapsulation → Private fields with getters/setters
- Abstraction → Service interfaces
- Inheritance → User → Student/Admin
- Polymorphism → Allocation strategies
