## 📖 Project Description

The Hostel Room Allocation System is a full-stack web application designed to manage hostel room allocation for students in an institution.

Students can apply for hostel accommodation, and administrators can assign rooms based on availability and allocation strategy.

The main focus of this project is backend system design using OOP principles, clean architecture, and design patterns.

---

## 🎯 Project Objectives

- Manage student registration and authentication
- Track hostel and room details
- Monitor room availability
- Allocate rooms using a defined allocation strategy
- Maintain allocation records
- Implement role-based access (Student / Admin)

---

## 🏗 System Architecture

The backend follows a layered architecture:

- Controller Layer → Handles API requests
- Service Layer → Business logic
- Repository Layer → Database operations
- Model Layer → Core entities

---

## 🧠 OOP Concepts Used

- Encapsulation → Data hiding within entities
- Abstraction → Service interfaces
- Inheritance → User → Student / Admin
- Polymorphism → Allocation strategy implementation

---

## 🎨 Design Pattern Used

### Strategy Pattern
Used in room allocation logic.

Different allocation strategies can be implemented such as:
- First Come First Serve
- Priority-based allocation

## 📂 Core Entities

## 🔄 Main Workflow

1. Student registers and logs in
2. Student applies for hostel accommodation
3. Admin reviews applications
4. System checks room availability
5. Room is allocated using allocation strategy
6. Allocation is recorded and status is updated
